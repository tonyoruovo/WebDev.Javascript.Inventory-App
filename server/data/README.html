The following were are models for a sample inventory app.

  1. Product model

    • Atrributes
      • Id
      • Name
      • Desc
      • Category
      • Manufacturer
      • Supplier info
      • Price
      • Quantity in stock
      • Reorder point
      • Barcode/UPC
    • Relationships
      • May have relationships with sales transactions, purchase orders, and suppliers.

  2. Location model

    • Atrributes
      • Id
      • Name
      • Address
      • Capacity
      • Account details
      • Ts & Cs
      • Signature
      • logo light
      • logo dark
      • Default status (e.g, Ordered, Confimred, Processing, Shipping, Delivered)
      • Codes of acceptable currencies
    • Relationships
      • Products can be stored at specific locations.

  3. Inventory Transaction model

    • Atrributes
      • Id
      • Product Id
      • Location Id
      • Quantity
      • Timestamp
      • User/Employee info
      • Type (e.g., purchase, sale, transfer)
    • Relationships
      • Links product changes to specific transactions.

  4. Supplier model

    • Atrributes
      • Id
      • Contact Info
      • Payment terms
    • Relationships
      • Products can be associated with specific suppliers.

  5. Customer model

    • Atrributes
      • Id
      • Contact Info
      • Transaction history
    • Relationships
      • Links customer data to sales transactions.

  6. Purchase Order model

    • Atrributes
      • Order number
      • Supplier Info
      • Order Date
      • Expected delivery date
      • Status (e.g., pending, received)
    • Relationships
      • Tied to products being ordered.

  7. Sales Order model

    • Atrributes
      • Order number
      • Customer Info
      • Order Date
      • Status (e.g., pending, received, cancelled, failed)
    • Relationships
      • Linked to products sold.

  8. User/Employee model

    • Atrributes
      • Id
      • Role/Permission
      • Login credentials
      • Status (e.g., pending, received, cancelled, failed)
    • Relationships
      • Associates transactions and changes with specific employees.

  9. Alerts and Notifications model

    • Atrributes
      • Id
      • Type (e.g., low stock, expired products)
      • Message
      • Timestamp
    • Relationships
      • Relates to products and inventory levels.

  10. Report model

    • Atrributes
      • Id
      • Type (e.g., sales report, inventory valuation)
      • Parameters and filters
      • Generated Date
    • Relationships
      • May be related to various data models for reporting purposes.

  11. Contact model

    • Atrributes
      • Id
      • Full name
      • Company name (if applicable)
      • Email
      • Phone
      • Address
      • Additional contact information (e.g., fax number, website)
    • Relationships
      • Associated with suppliers, customers, employees, and possibly other entities like manufacturers or distributors.
      • The Contact Model allows you to centralize and manage contact information for various entities that interact with your inventory management system. This can help in tracking and communicating with suppliers, customers, and employees efficiently. Each contact can be associated with relevant entities in the system, such as linking a contact to a specific supplier or customer.
